viewWorkInProgressTapped:revisionId:

Called if the end user taps “VIEW ON BEHANCE” in the success popup, indicating that they would like to view the published work in progress if this method is not handled in the delegate, the project will be opened in the Behance app, or, if it’s not installed, on behance.net in Safari.

- (void)viewWorkInProgressTapped:(NSNumber *)wipId revisionId:(NSNumber *)revisionId

Parameters

wipId

the id of the work in progress that was created

revisionId

the id of the work in progress revision that was created

Discussion

Called if the end user taps “VIEW ON BEHANCE” in the success popup, indicating that they would like to view the published work in progress if this method is not handled in the delegate, the project will be opened in the Behance app, or, if it’s not installed, on behance.net in Safari.

wipPublishShouldPresentSuccessPopup

Called immediately after wipPublishDidComplete:revisionId: allows delegates to override the default behavior of presenting a popup window upon publishing success

- (BOOL)wipPublishShouldPresentSuccessPopup

Return Value

delegates should return NO if they wish to prevent the popup window from being shown, default is YES

DEPRECATED: If a success pop up is required, you must use wipPublishDidComplete:(NSNumber )wipId revisionID:(NSNumber )revisionId instead. Otherwise, returning NO for this method is still supported.

Discussion

Called immediately after wipPublishDidComplete:revisionId: allows delegates to override the default behavior of presenting a popup window upon publishing success

Supply a custom string to use when sharing work in progress to external services (ex. Twitter). Please note that AdobePublishAccountTypeFacebook is not supported.

Return Value

Discussion
