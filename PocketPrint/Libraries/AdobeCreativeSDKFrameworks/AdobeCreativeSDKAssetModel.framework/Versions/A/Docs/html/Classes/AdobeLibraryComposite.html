AdobeLibraryComposite Class Reference

Inherits from NSObject
Declared in AdobeLibraryComposite.h

Overview

This is the main class that represents a library aka library composite. A library composite contains elements which are represented by AdobeLibraryElement.

Library creation and deletion should be done via AdobeLibraryManager. See newLibraryWithName and deleteLibraryWithId on AdobeLibraryManager.

getRenditionPath:withSize:isFullSize:handlerQueue:onCompletion:onError:

Get the path to an element’s rendition for a particular size. This method will return a rendition in the following order:

1) First check the local cache for a rendition of the requested size. 2) Next it will check for a representation that has been marked as a rendition with the given size. 3) After it will find any representation that can be used to generate a rendition and generate one using Adobe’s imaging service. 4) Returns NO if all the above methods failed to find a suitable rendition.

- (BOOL)getRenditionPath:(NSString *)elementId withSize:(NSUInteger)size isFullSize:(BOOL)fullSize handlerQueue:(NSOperationQueue *)queue onCompletion:(void ( ^ ) ( NSString *))completionBlock onError:(void ( ^ ) ( NSError *))errorBlock

Parameters

elementId

the id of the element for which to get the rendition.

size

the size of the requested rendition (ignored if fullSize is specified)

fullSize

whether or not the full size rendition is requested

queue

Optional parameter. If not nil determines the operation queue completion and error blocks get executed on.

completionBlock

The block that gets called with the file path in case of success.

errorBlock

The block that gets called with the error object in case of error.

Return Value

YES if this rendition can be found or generated for the element, otherwise NO.

Parameters

fullSize
