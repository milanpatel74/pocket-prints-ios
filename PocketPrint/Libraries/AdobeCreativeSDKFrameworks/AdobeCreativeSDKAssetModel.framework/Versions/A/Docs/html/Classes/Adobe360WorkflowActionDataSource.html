Instance Methods

actionsToDisplay

The method returns an array of actions to be displayed. These actions initiate the primary app workflow.

- (NSArray *)actionsToDisplay

Return Value

An array of Adobe360WorkflowAction instances.

Discussion

The method returns an array of actions to be displayed. These actions initiate the primary app workflow.

These actions are filtered and sorted based on the following algorithm: - the actions will be split into generic actions, or actions without specified subtypes, and the actions with subtypes specified by [Adobe360WorkflowActionDataSource setWhitelistedSubTypes:]. - these groups are sorted by priority using the block specified at [Adobe360WorkflowActionDataSource setPrioritizeBlock:] - the two groups are joined with the specific actions prioritized above the generic list - all actions are sorted by usage. This usage is increased using [Adobe360WorkflowActionDataSource increaseFrequencyForAction:]

The method loads the action registry from the server. The registry is locally cached as well as a version of it is included in the bundle. Any network error returns the cached registry if one is is available. Calls both willLoadActionRegistry & didLoadActionRegistry delegate methods.
