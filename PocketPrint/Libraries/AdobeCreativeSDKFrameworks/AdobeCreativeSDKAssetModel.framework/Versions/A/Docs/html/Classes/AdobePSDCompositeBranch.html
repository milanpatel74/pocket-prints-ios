Inherits from NSObject
Declared in AdobePSDCompositeBranch.h

Overview

AdobePSDCompositeBranch represent a branch of a PSD composite. PSD composites support multiple branches. Developers work with a current branch but have the ability to bring in other branches that can then be merged in.

The following properties/methods have been deprecated:

  • -getLayersIntersectingPath:withLayerComp:
  • –layerIsVisible:withLayerComp:
  • –layerHasVisibleChildren:ofTypes:withLayerComp:recursively:

layerHasVisibleChildren:ofTypes:withLayerComp:recursively:

Returns whether a layer contains any visible sub-layers, optionally taking into account the effects of an applied layer comp.

- (BOOL)layerHasVisibleChildren:(AdobePSDLayerNode *)parentLayer ofTypes:(AdobePSDLayerNodeType)typesMask withLayerComp:(AdobePSDLayerComp *)layerComp recursively:(BOOL)recursive

Parameters

parentLayer

Must be either an existing layer node of type AdobePSDLayerNodeTypeGroup or nil in which case the check will start at the root.

typesMask

Bit mask specifying which layer types should be checked for visibility. AdobePSDLayerNodeTypeGroup bit mask will be ignored if recursive is true.

layerComp

The optional layer comp to apply when checking for visibility.

recursive

Whether the check should recursively include the result from sub layers.

Return Value

True if any sub-layers of the specified types are visible.

Discussion

Returns whether a layer contains any visible sub-layers, optionally taking into account the effects of an applied layer comp.

Note that, when performing a recursive check, if a sub-layer group is invisible, then all of its children are considered invisible regardless of their visibility settings. If a recursive check is not performed, the layer group’s visibility is included in the check only if provided as part of the bit mask of types to consider.
